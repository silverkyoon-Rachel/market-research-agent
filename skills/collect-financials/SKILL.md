# 재무제표 수집 + AI 심층 분석 스킬

이 스킬은 기업의 재무제표 데이터를 수집하고, AI로 주석·비용·투자 분석까지 포함한 심층 인사이트를 도출합니다.

## 수집 대상 데이터

- 매출액 (Revenue)
- 영업이익 (Operating Income)
- 당기순이익 (Net Income)
- 총자산 (Total Assets)
- 총부채 (Total Liabilities)
- 자본총계 (Total Equity)
- 영업이익률, 순이익률, ROE, 부채비율 (계산)

## API 키 조회 순서

API 키는 다음 순서로 확인합니다:
1. `backend/.env` 파일 (프로젝트 환경변수) — Read 도구로 파일을 읽어 해당 키 값을 확인
2. `.claude/market-research-agent.local.md` (플러그인 설정) — 1번에 없을 경우 확인

키 매핑:
| backend/.env | local.md | 용도 |
|---|---|---|
| `DART_API_KEY` | `dart_api_key` | DART 재무제표 |
| `FSC_API_KEY` | — | 금융위원회 기업재무정보 |

## 수집 분기 로직

### 1. 국내 기업 (상장/비상장 모두)

DART API를 우선 사용합니다. **DART API 키 필요.**

**수집 절차:**
1. 위 API 키 조회 순서에 따라 `DART_API_KEY` 확인
2. 키가 있으면:
   - DART API `/fnlttSinglAcnt.json`으로 재무제표 조회
   - 최근 3개년 연간 + 최근 분기 데이터 수집
   - `backend/collectors/dart_collector.py`의 `DartCollector` 활용
   - 엔드포인트: `POST /api/settings/dart/sync` (기업 목록), DART API 직접 호출
3. 키가 없으면:
   - WebSearch로 "{기업명} 재무제표 매출액 영업이익 {현재연도}" 검색
   - WebFetch로 DART 공시페이지, 네이버 금융 종목분석, FnGuide 등에서 수집
   - 공공데이터포털 금융위원회 기업재무정보 API 시도 (별도 키 필요 시 안내)

**비상장 외감법인:**
- DART에 공시 의무가 있으므로 DART API로 동일하게 수집 가능
- 종목코드 없이 기업고유번호(corp_code)로 조회

**비상장 비외감법인:**
- DART에 데이터 없음
- WebSearch로 대체: "{기업명} 매출 영업이익", 기업 IR페이지, 언론보도 등에서 수집

### 2. 미국 기업

SEC EDGAR를 우선 사용합니다. **API 키 불필요.**

**수집 절차:**
1. WebSearch로 "{company name} SEC EDGAR 10-K filing" 검색
2. SEC EDGAR에서 10-K (연간) 또는 10-Q (분기) 보고서 확인
3. WebFetch로 EDGAR XBRL 재무데이터 페이지 접근
4. 매출, 영업이익, 순이익, 자산, 부채 추출

**대체 경로:**
- WebSearch로 "{company name} annual revenue net income {year}" 검색
- Bloomberg, Reuters, Macrotrends 등에서 재무 요약 수집

### 3. 영국 기업

Companies House를 우선 시도합니다. **API 키 무료 발급.**

**수집 절차:**
1. API 키 조회 순서에 따라 `companies_house_api_key` 확인
2. 키가 있으면 → Companies House API로 직접 조회
3. 키가 없으면 → WebSearch로 "{company name} Companies House accounts" 검색
4. WebFetch로 Companies House 공시 페이지에서 재무 데이터 추출

### 4. 기타 해외 기업

WebSearch를 통한 유연한 탐색을 합니다.

**수집 절차:**
1. WebSearch로 다음을 순차 검색:
   - "{company name} financial statements {year}"
   - "{company name} revenue operating income annual report"
   - "{company name} investor relations financial data"
2. 검색 결과에서 가장 신뢰할 수 있는 소스 선택:
   - 기업 공식 IR 페이지 (최우선)
   - 각국 기업등기소/공시시스템
   - Bloomberg, Reuters, Financial Times 등 금융 매체
   - 업계 리서치 보고서
3. WebFetch로 해당 페이지에서 재무 수치 추출
4. 출처와 데이터 기준일을 반드시 명시

### 5. 어디서도 못 찾은 경우

모든 경로가 실패하면:
- "이 기업의 공개된 재무정보를 찾지 못했습니다"
- "다음 방법으로 데이터를 추가할 수 있습니다:"
  - IR 자료/사업보고서 파일을 제공해주세요
  - 알고 있는 재무 수치를 직접 입력해주세요
  - DART API 키를 설정하면 국내 외감법인 데이터를 수집할 수 있습니다 (/setup-keys)

## 빠른 실패 규칙

수집 시 아래 규칙을 반드시 준수합니다:

1. **시도 횟수 제한**: 1차 소스 1회 + 대체 소스 최대 2회 = 총 3회 이내
2. **1차 소스 실패 시**: DART API 실패 → WebSearch 1회 → WebFetch 1회 → 그래도 없으면 "수집 실패" 반환
3. **WebFetch 제한**: 최대 2개 URL까지만 시도. 응답 없거나 파싱 불가 시 즉시 포기
4. **부분 수집 허용**: 3개년 중 일부만 있어도 OK. 매출만 있고 영업이익이 없어도 수집된 것만 반환
5. **실패 반환 형식**: `⚠️ 수집 실패: {사유}. 시도한 소스: {소스 목록}`

**수집 우선순위 (국내 기업 예시):**
```
DART API → (실패) → WebSearch "{기업명} 재무제표 매출액" → (실패) → WebFetch 네이버금융 1회 → (실패) → 수집 실패 반환
```

## AI 심층 분석 (수집 후 반드시 수행)

수집된 재무제표 데이터를 기반으로 Claude가 직접 다음 심층 분석을 수행합니다.

### 분석 항목

**1. 성장성 분석**
- 매출 성장률 YoY (3개년 추세: 가속/둔화/역성장)
- 영업이익 성장률 YoY
- 순이익 성장률 YoY
- 성장 드라이버 추정 (매출 증가 vs 마진 개선 vs 양쪽 모두)

**2. 수익성·비용 구조 분석**
- 영업이익률 추이 (개선/악화 판단과 원인 추정)
- 순이익률 추이
- 매출원가율 vs 판관비율 변화 (가능한 경우)
- 영업이익과 순이익 괴리가 큰 경우 → 영업외손익(금융비용, 환차손 등) 분석
- 비용 증가가 매출 증가를 초과하는지 여부

**3. 재무 건전성 분석**
- 부채비율 추이 (안정/악화)
- 유동비율 추정 (가능한 경우)
- 순차입금 규모 추정
- 이자보상배율 추정 (영업이익 ÷ 금융비용)
- 자본잠식 위험 여부

**4. 투자 활동 분석** (주석/추가 데이터 확보 시)
- CAPEX 규모 및 추이 (설비투자 확대/축소)
- R&D 투자 비중 (있는 경우)
- 인수/매각 활동 (현금흐름표 또는 뉴스에서 확인)
- WebSearch로 "{기업명} 설비투자 CAPEX" 또는 "{기업명} 투자 계획" 검색하여 보강

**5. 주석 기반 숨은 인사이트** (DART/EDGAR 상세 보고서 확인 시)
- 우발채무, 소송 리스크
- 특수관계자 거래 규모
- 재고자산 변동 (재고 급증 → 수요 둔화 시그널)
- 매출채권 회전율 변화 (현금회수 속도)
- 리스부채 규모 (IFRS16)

**6. M&A 관점 재무 매력도**
- 순자산가치(BPS) 대비 시장 평가
- 안정적 현금 창출력 여부
- 부채 인수 시 부담 규모
- 인수 후 시너지 가능 영역 (비용 절감, 매출 확대)

### 분석 방법
- 수집된 3개년 재무데이터의 추이를 정량적으로 분석
- 비율/성장률은 직접 계산하여 제시
- 주석 데이터가 필요한 경우 WebSearch로 "{기업명} 사업보고서 주석" 또는 DART 공시 페이지를 추가 확인
- 업종 평균 비교가 필요한 경우 WebSearch로 동종업계 데이터 수집
